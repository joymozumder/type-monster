Question & Answer

  1. Question: What are the differences between Local Storage and Session Storage?

    Answer:
    Local Storage: Local storage is useful for storing data that the user will need to access later, such as offline data. The stored data is not deleted when the browser is closed. It allows 10MB of data to be stored.

    Session Storage: Session storage is useful for storing data that is sensitive, such as login credentials. The data is stored only for the session and will be deleted when the browser is closed. It allows 5MB of data to be stored.

  2. Question: What are the differences between Global Scope and Block Scope?

    Answer:
    Global Scope: Variables declared Globally (outside any function) have Global Scope. Global variables can be accessed from anywhere in a JavaScript program. Variables declared with var, let and const are quite similar when declared outside a block.

    Block Scope: Block scope restricts the variable that is declared inside a specific block, from access by the outside of the block. The let & const keyword facilitates the variables to be block scoped. Variables declared with the var keyword, do not have block scope.

  4. Question: How many ways can we get undefined?

    Answer:
    There are 8 ways to get Undefined:

    1. Variable that is not initialized will give undefined.
    2. Function with no return.
    3. Parameter that is not passed will be undefined.
    4. If return has nothing on the right side will return undefined.
    5. Property that doesn't exists on an object will give you undefined.
    6. Accessing array elements outside of the index range.
    7. Deleting an element inside an array.
    8. Set a value directly to undefined.
